# FacilityTrack — Aquatic Facility Operations Management

## Overview

FacilityTrack is a full-stack web application designed for aquatic facilities (pools, spas, steam rooms) to manage operations and ensure compliance with NZS 5826:2010 standards. It provides tools for water quality testing, compliance reporting, maintenance scheduling, asset management, and staff administration. The project aims to streamline facility management, enhance safety, and ensure regulatory adherence, with a future vision for a desktop application.

## System Architecture

FacilityTrack is built as a monorepo using pnpm workspaces.

**Technology Stack:**
- **Monorepo:** pnpm workspaces
- **Backend:** Node.js 24, Express 5, Drizzle ORM, `express-session`, `bcryptjs`.
- **Frontend:** React 19, Vite, Tailwind CSS v4, Wouter for routing, shadcn/ui.
- **Database:** SQLite with `better-sqlite3` v12 via Drizzle ORM for local and desktop deployments. PostgreSQL for multi-client Azure deployments.
- **API:** OpenAPI 3.0 specified in `lib/api-spec/openapi.yaml`, with client-side React Query hooks generated by Orval.
- **Validation:** Zod and `drizzle-zod`.

**Core Features:**
- **Authentication:** Session-based authentication with email/password and PIN login, utilizing `express-session` and `bcryptjs`. Includes RBAC with roles: `superuser`, `admin`, `user`.
- **NZS 5826:2010 Compliance Logic:** Implemented in `artifacts/api-server/src/lib/compliance.ts`, this module computes compliance results based on predefined pool and spa thresholds for chemical levels, pH, temperature, and turbidity.
- **Licensing Architecture (Desktop App):** A dual-component system for per-PC licensing: a standalone Licence Server (`artifacts/license-server`) and an in-app license client. It includes offline grace periods and a `LicenseGate` component in the frontend to manage access based on license status.
- **Desktop Conversion:** The application is packaged as a single-tenant Windows desktop app using Electron, `electron-builder`, and `electron-updater`. The desktop version embeds the API server and bundles the React frontend.
- **OneDrive Nightly Backup:** For desktop users, an in-process backup mechanism leverages Microsoft Graph API to securely back up the SQLite database to OneDrive, supporting OAuth 2.0 device-code flow for connection.
- **UI/UX Design:** Features an oceanic theme with primary (HSL 200 90% 40%), sidebar (HSL 222 47% 11%), and accent (HSL 199 89% 48%) colors, implemented with Tailwind CSS and shadcn/ui components.

**Database Schema:**
The system uses 18 tables to manage various aspects of facility operations including:
- `facilities`, `pools`, `testResults`, `waterBalanceTests`, `poolClosures`
- `steamRoomChecks`, `workOrders`, `assets`, `maintenanceSchedules`
- `staff`, `staffQualifications`, `trainingRecords`, `notifications`
- `appUsers`, `complianceDocuments`, `systemSettings`, `licenseState`, `auditLogs`.

**API Structure:**
All API routes are mounted under `/api`, covering authentication, CRUD operations for facilities, pools, test results, staff, assets, work orders, and dedicated endpoints for dashboard summaries and audit logs.

- **API Generation:** Orval for generating React Query hooks from OpenAPI specifications.
- **Desktop Packaging:** Electron, `electron-builder`, `electron-updater`.
- **Cloud Services:** Azure Container Apps, Azure PostgreSQL (for multi-client hosting), Microsoft Graph API for OneDrive integration.
